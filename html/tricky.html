HTML Tricky Concepts

Introduction

HTML is a markup language that is used to create the structure of web pages. It is important to understand the basic and tricky concepts of HTML in order to create well-structured and accessible web pages.

Creating a Text File in MacOS TextEdit

Step 1

Open TextEdit application on your macOS. It can be found using Finder then searching through the Applications Folder.

Step 2

Once you Open TextEdit, click the New Document button. If you see a text formatting menu and a tab ruler, you are in rich-text mode. If not, you are in plain-text mode.

Step 3

Close the document and head towards the top of your macOS system. Next to the Apple symbol, the TextEdit tab should be next to it. Hover over the TextEdit and click on Settings.

Step 4

On the New Document tab click on plain text. Under Options uncheck all of the automatic formatting options. On the Open and Save tab, select Display HTML files as HTML Code 5. Next, deselect "Add '.txt.' extension to plain text files."

Step 5

The rest of the defaults should be good as they are. When you are done, click on the red button of the settings box.

Step 6

Finally, you can click on the File tab next to TextEdit and click New to create a New Document. The original settings are reverted to the new changes.

Note: You can convert the document back to rich-text mode when you are not using TextEdit for HTML.

Why do we change the settings for TextEdit?

Changing the settings for TextEdit allows for proper rendering of the code on the website. The default settings are not suitable for HTML coding. The settings we changed allow for the code to be displayed as it is without any formatting or changes.

Creating a Text File in Windows' Notepad

Step 1

Open Notepad to create a new document window. For quicker access, use the search feature in your applications folder.

Step 2

Once you have opened Notepad a new untitled page will appear. If you navigated to a previously used page, click the add button to open up a new untitled page.

Step 3

Begin writing HTML in the page. Remember to use the basic document structure. Refer to HTML Main Guide for help.

Step 4

Click the File tab to open a dropdown menu. Find Save As and click on it to save the document.

Step 5

In the Save As window, select the location you want to save the file to then name the file. Remember to add the .html extension at the end of the file name. This is important because without the extension, the file will not be recognized as an HTML file. Be sure to select All Files in the Save as type dropdown menu and check the encoding is set to UTF-8. Click Save.

Step 6

To open the file in a web browser, navigate to the location where you saved the file. Double-click on the HTML file to open into your default browser or right-click on the file and select Open with to choose your preferred web browser from the list.

Document Validation

Document validation is the process of checking the markup of a document to ensure that it is valid and follows the rules of HTML. This is important because invalid markup can lead to rendering issues and accessibility problems. Here are common mistakes to check for:

  • DOCTYPE declaration
    • Without the declaration, the browser may not render the page properly to the correct version of HTML.
  • Incorrect tags
    • Missing or incorrect closing tags.
  • Nesting errors
    • Nesting lists tags.

Files and Folder Organization

Organizing your files and folders is important for keeping your project structured and easy to navigate. Here are some tips for organizing your files and folders:

  • Use descriptive names for your files and folders.
  • Use a consistent naming convention.
  • Use names without spaces for your files and folders.
    • To avoid spaces, you can use snake case (like_this), kebab case (like-this), camel case (likeThis), or Pascal case (LikeThis).
  • Group related files together in folders.
  • Keep your project folder clean and free of unnecessary files.
  • Use subfolders to organize your files further.

File and folder organization helps you use the correct relative path to link images, stylesheets, or scripts to your HTML document. This is important because if the file path is incorrect, the browser will not be able to find the file and it will not be displayed on the page. It may be helpful to create a sitemap of your website to organize your files and folders in the correct location.

Special Characters

HTML uses special characters to represent certain symbols and characters that have a special meaning in HTML. These characters are called character entities or escape characters. Here are some common character entities:

  • &lt; - less than
  • &gt; - greater than
  • &amp; - ampersand
  • &quot; - double quote
  • &#39; - single quote
