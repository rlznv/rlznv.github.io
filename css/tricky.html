CSS Tricky Concepts

Introduction

Units of Measurement

CSS uses units of measurement to define the size of elements on a page. There are two types of units: absolute and relative. It is recommended to use relative units rather than absoulte as relative units provide more adaptability, flexibility, and responsiveness to the design.

Absolute Units

Absolute units are fixed units and do not change with the size of the viewport. They are not recommended for responsive design.

px = pixel, defined as equal to 1/96 of an inch in CSS3
in = inches
mm = millimeters
cm = centimeters
q = ¼ millimeters
pt = points (1/72 inch). Points are a unit commonly used in print design
pc = picas (1 pica = 12 points of 1/6 inch). Picas are a unit commonly used in print design.

Relative Units

Relative units are based on the size of the viewport or the size of the parent element. They are recommended for responsive design.

em = a unit of measurement equal to the current font size.
ex = x-height, approximately the height of a lowercase “x” in the font
rem = root em, equal to the em size of the root element (html)
ch = zero width, equal to the width of a zero (0) in the current font and size
vw =viewport width unit, equal to 1/100 of the current viewport (browser window) width
vh =viewport height unit, equal to 1/100 of the current viewport height.
vmin = viewport minimum unit, equal to the value of vw or vh, whichever is smaller.
vmax = viewport maximum unit, equal to the value of vw or vh, whichever is larger

It is important to be able to distingush the difference between the unit rem and em. Rem comes from the root element and em comes from the parent element.

Media Queries

Media queries are used to apply different styles to different screen sizes. They are used to create responsive designs that adapt to different devices. Media Queries are conditional meaning they are only applied if the condition is met. The most common use of media queries is to change the layout of a page based on the size of the viewport. It can also allows a print attribute which adapts your web page to a print versions based on your specifications. Here is the structure of a media query: 

@media type and (feature: value) {
      /* CSS rules go here */
    }

Here is a simple example of the media query being used:

@media screen and (orientation: portrait) {
      body {background-color: lightblue;}
     }

Media Print

Media print is used to apply styles when the page is printed. It is used to create a print version of a page that is different from the screen version. Here is an example of media print:

@media print {
      /* CSS rules go here */
     }

CSS Box Model

The CSS box model is a box that wraps around every HTML element. The model is used to created definiton and proper structure to the order in which HTML elements are organized. It consists of the following components:

  • Content: The innermost part of the box, where text and images appear.
  • Padding: The space between the content and the border. It is transparent and can be set to different values for each side.
  • Border: The line that surrounds the padding and content. It can be styled with different colors, widths, and styles.
  • Margin: The outermost space that separates the element from other elements. It is also transparent and can be set to different values for each side.

CSS Floating

Floating is a CSS property that allows elements to be positioned to the left or right of their container. It can be used to create layouts in which elements can flow around each other. The float property can take the following values:

  • Left: The element will float to the left of its container.
  • Right: The element will float to the right of its container.
  • None: The element will not float and will be positioned normally in the document flow.
  • Inherit: The element will inherit the float property from its parent element.

CSS Positioning

Having each of the positioning methods has its own purpose. By allowing the element to modify how the positioning works we can make them behave differently.

Static

With static positioning it will appear as it normally would in a normal document flow.

position: static;

Relative

Relative positioning moves the box relative to the original position of the document flow. The space in which the positioning would be occupied as will now be an empty space.

position: relative;

Absolute

With absolute position elements, they are removed from the document flow. The space they occupy is closed, they have no influence on the layout of any surrounding elements.

position: absolute;

Fixed

Fixed positioning has the element fixed in one position of the viewport even when the user is scrolling. This element is removed from the document flow and is relative to the viewport.

position: fixed;

Sticky

With sticky positioning, it combines relative and fixed. It stays relatively positioned until it is scrolled into a position relative to the viewport, this is when it remains fixed. Sticky stays inline and scrolls with the document if the browser does not support it.

position: sticky;
