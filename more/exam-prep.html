Exam Help

Introduction

We created our Exam Help page to provide students with practice questions that reflect the concepts tested in the midterm exam. The midterm exam primarily tests foundational web development knowledge. This knowledge includes modern HTML5 best practices, how a browser interacts with a server, and the role of JavaScript on the web. In addition to that foundational knowledge, students will also need to draw and/or describe any given HTML element or CSS rule.

Here's a brief summary what you'll need to pratice to do well on the exam.

  • Interpreting HTML markup and CSS rules, then drawing and/or describing it.
  • Identifying different HTML, CSS, and JavaScript elements, then explaining what those elements do on a web page.
  • Differentiating between block and inline elements.
  • Explaining the importance of CSS properties like display.
  • Describing the necessary HTML elements for a boilerplate HTML5 web page.
  • Explaining how attributes, properties, and values affect HTML elements.

Answers to our practice questions are blurred by default. Click on the blurred text and images to reveal the answers.

Draw and Describe

Practice interpreting and drawing HTML markup and CSS rules. These questions will primarily test your knowledge of how CSS affects the appearance of HTML elements.

The HTML and CSS for some questions are formatted for readability, but not all of them. This is intentional and will prepare you for the exam's formatting (neat formatting is for human readability, not machines!).

Q1 Draw and describe how this HTML and CSS would appear on a web page. 

<div>
  <span id="one">A</span>
  <span id="two">B</span>
  <span id="three">C</span>
  <span id="four">D</span>
  <span id="five">E</span>
</div>
html {
  font-size: 24px;
} 
span {
  margin: 0;
  pading: 0;
  border: .25rem dashed red;
} 
div {
  display: grid; 
  width: 75%; 
  margin: 0 auto; 
  color: #660066; 
  grid-template-columns: 1fr 1fr 1fr 1fr; 
  grid-template-rows: 2fr 2fr; 
  grid-template-areas: "one three three four" "two two five four";
  gap: 5px;
} 
#one {
  grid-area: one;
} 
#two {
  grid-area: two;
}
#three {
  grid-area: three;
} 
#four {
  grid-area: four;
} 
#five {
  grid-area five;
} 
@media print {
  div, span {
    display: block;
  }
}

The <div> element's <display> property is set to grid, switching its display to CSS grid. The CSS rules produce a grid with four columns and two rows. The grid's width is 75% of its container. The grid is also horizontally centered in its container and have 5 pixel gaps between each grid area.

The <div>'s color property affects the foreground color of the <div>'s child elements, meaning that the <span>'s text is dark purple.

The <span> elements have a red dashed border of .25rem or 6 pixels. Each <span> element has a rule that dictates its grid area. The <span> elements have their margin and padding set to 0, meaning that each <span>'s text is in the very top left corner.

There is a media query that controls how the <div> and <span> elements display when printed. When printed, they will appear as block elements, creating a column.

Q2 Draw and describe how this HTML and CSS would appear on a web page. 

<aside>
  <span id="one">1</span>
  <span id="two">2</span>
  <span id="three">3</span>
  <span id="four">4</span>
  <span id="five">5</span>
</aside>
html { font-size: 24pt; } span { margin: 0; border: 3px solid black; } aside { display: grid; width: 50%; margin: auto; color: #fafafa; grid-template-columns: repeat(3, 1fr); grid-template-rows: repeat(3, 1fr); grid-template-areas: "one one one" "two three four" "two five five"; } #one { grid-area: one; } #two { grid-area: two; } #four { grid-area: four; } #three { grid-area: three; } #five { grid-area: five; } @media screen and (max-width: 320px) { aside { display: none; } }

The <aside> element's <display> property is set to grid, switching its display to CSS grid. The CSS rules produce a grid with three columns and three rows. The grid's width is 50% of its container. The grid is also horizontally and vertically centered in its container.

The <div>'s color property is set to #fafafa, a color that's nearly white, meaning that the <span> elements' text is nearly invisible on a white background.

The <span> elements have a solid black border of 3 pixels. Each <span> element has a rule that dictate its grid area, though it is hard to see each <span>'s text.

Refer to the image below for the appearance of the <aside> when seen on the web with a white background.

There is a media query that controls the display of the aside element. When the viewport is less than 320 pixels wide, the aside element will not display.

Q3 Draw and describe how this HTML and CSS would appear on a web page. 

<nav class="menu"><p>Hello and welcome to Bobson Dugnutt's website!</p></nav>
nav { position: fixed; top: 0; left: 0; width: 100%; color: #ffffff; background-color: blue; font-size: 12pt; font-family: Charter, serif; } .menu { font-size: 36pt; padding: 1rem; } .menu p { font-size: 48pt; text-align: center; }

The <nav> container is fixed to the top left of the viewport, spanning the viewport's entire width.

The <nav>'s foreground color is set to pure white, and its background color is set to blue. Therefore, the <p> element within the <nav> container will display as white text.

The <p>'s text will appear center aligned, with 1 rem of padding, in the Charter font (with the browser's default serif font as a fallback), and in a 48pt font size.

The .menu selector adds a new padding rule of 1rem. The .menu selector is more specific than the nav selector, meaning .menu's 36pt font size overrides nav's 12pt font size. However, the .menu p selector is the most specific selector, overriding all other rules. This results in the final font size of the <p> being 48pt.

Q4 Draw and describe how this HTML and CSS would appear on a web page. 

<a href="/home.html" class="button">Click me!</a>
a.button { display: block; width: 5rem; font-size: 1rem; padding: 1rem; border: 2px solid purple; border-radius: 5px; text-transform: uppercase; transform: rotate(20deg); transition: all .5s; } a.button:hover { border-radius: 20px; transform: rotate(0deg); transition: all .5s; }

The HTML creates an <a> element that displays the text, "Click me!"

The CSS styles the <a> to display as a block element button with hover effects. The text size of the button is 1 rem with 1 rem of padding on all sides. The text is also transformed to be in all uppercase.

In its default state, the <a> element has a width of 5 rem, a 2 pixel solid purple border, 5 pixel rounded corners, and is rotated 20 degrees.

Upon hovering on the <a> element, the element's rotation is set to the default of 0 degrees and its corners are rounded to 20 pixels. The transition between the default and hovered states is set to half a second for all properties, resulting in a smooth transition animation between states.

Q5 Draw and describe how this HTML and CSS would appear on a web page. 

<article><p id="one">1</p><p id="two">2</p><p id="three">3</p><p id="four">4</p><p id="five">5</p></article>
article {
  display: flex;
  flex-flow: column-reverse;
  border: 5px solid #00ff00;
  width: 50%;
  margin: auto;
  justify-content: center;
  height: 500px;
}
    
p {
  margin: 0;
  text-align: center;
  padding: 0;
  border: 1px solid black;
}
    
#three {
  order: 5;
}

The <article> element is a flex container that is center aligned in its parent container with a 5 pixel solid green border, a width of 50% of its container, and a height of 500 pixels.

The <article> container has five flex items. These five items are arranged in a column and flow in reverse order. The items are center justified on the flex container's main axis (the Y-axis), meaning that they are vertically aligned center.

The <p> flex items have center aligned text and a 1 pixel solid black border.

The <p> element with an id of three has its order set to 5, meaning that it has the highest value in regards to order. Since the flow of the <article> is set to reverse, the <p> with the id of three is the top-most item.

Element Identification

Practice identifying and describing the function of HTML, CSS, and JavaScript elements. These questions will primarily test your knowledge of HTML elements and their attributes.

Q1 Identify the element type and describe its function on a web page. 

<script src="/js/darkmode.js"></script>

This is an HTML element that imports a JavaScript file into the document. The src attribute tells the server to look in the server's root folder for the "js" folder, then find the "darkmode.js" file.

Q2 Identify the element type and describe its function on a web page. 

<p>こんにちは皆さん。元気ですか。このウェブサイトが好きですか。<p>

If the UTF-8 (Unicode Transformation Format 8-bit) character set is declared in the head of the HTML page, this paragraph text element will display text in Japanese.

If UTF-8 is not declared in the head of the document, the text may not display correctly.

Q3 Identify the element type and describe its function on a web page. 

<p style="font-family: serif; font-size: 2rem; margin: .5rem 0 1rem 0;">

This is the opening tag of a paragraph text element with inline CSS.

The text in the paragraph will appear in the browser's default serif font. The text will be twice the size of the font size declared in the document's root (or twice the size of the browser's default font size if the font size is not declared in the document's root).

The paragraph element will have a top margin of .5 rem, a right margin of 0, a bottom margin of 1 rem, and a left margin of 0.

Q4 Identify the element type and describe its function on a web page. 

nav { display: grid; grid-template-columns: 1fr 1fr 1fr 2fr; }

This is a CSS rule that sets the <nav> element's display to a grid with four columns. The widths of the columns are defined using fractional units.

The columns will adjust their widths to maintain the proportions of 1:1:1:2 in the grid's available space. In this scenario, because frs are the only units used, the widths of the columns will be equivalent to 20%, 20%, 20%, and 40%. Keep in mind though, percentages are not the same as fractional units.

Q5 Identify the element type and describe its function on a web page. 

let theme = document.getElementById("theme");

This is a JavaScript variable declaration. The variable theme represents the HTML element with the id of theme.

Q6 Identify the element type and describe its function on a web page. 

<link rel="icon" type="image/x-icon" href="/img/favicon.ico">

This is an HTML element that links to the web page's favicon, the image that appears as the tab's icon.

The rel attribute indicates the relation of the linked element to the web page. The href tells the server that "favicon.ico" is located in the "img" folder of the root folder.

Short-Essay Questions

Practice writing short paragraphs explaining crucial HTML, CSS, and JavaScript concepts. These questions will test your knowledge of modern best practices and web development essentials.

Q1 Describe what a boilerplate HTML document is, what's typically in one, and why a web developer may use one.

A boilerplate HTML document is a essentially a standard, unfilled template or "frame" of an HTML document. It contains necessary information like the document type declaration and the <html>, <head>, and <body> tags. There is no set standard for HTML boilerplates, so some may contain information like character set declarations, viewport declarations, favicon links, or links to CSS or JavaScript libraries.

A web developer may use boilerplate HTML documents to speed up their web development process. Using a standardized boilerplate tailored for a specific project allows a developer to save time and more easily create uniform web pages for a website.

Q2 Describe the difference between inline and block elements. Explain how a web developer may be able to control the display of block and inline elements.

Block elements are HTML elements that, by default, take up the entire width of their container, forcing following elements onto a new line. Common block elements include: <p>, <div>, <section>, and <nav>.

Inline elements are essentially the opposite of block elements. They are HTML elements that, by default, take up only the required width of its content and do not force following elements onto a new line. Common block elements include: <a>, <em>, <strong>, and <span>.

A web developer can control the display of elements using the CSS display property. For example, a web developer may set the display of an <a> element to "block."

Q3 Describe how a web developer may use CSS flexbox to create a web-based menu.

CSS flexbox is best used for content that is arranged to flow in one direction. CSS flexbox may be used in this scenario to create a row or column based menu.

A single-column flexbox would be a simple flexbox-based solution for a menu. A web developer could create a flex container with items category sections. The flex container's flex direction would be set to "column" and the content's justification could be set to "flex-start," creating a simple single-column layout.

To add another more columns, a developer could set the flex container's wrapping behavior to "wrap" and set the height on the flex container. A set height that forces the columns to wrap would create multiple columns.

Q4 Describe how a web developer may use CSS grid to create a web-based menu.

CSS grid is best used for content that needs to be flow or be assigned to a certain grid layout. CSS grid may be used in this scenario to create a grid-based menu.

A picture-based four-column grid would be a simple grid-based solution for a menu. A web developer could create a grid container with four columns and one row to start, then set up the grid to automatically generate new cells as new grid items are added to the grid.

Each grid item could be a container that consists of a picture of the menu item, the menu item's name, and the menu item's price. Additionally, when adding new items, the web developer could let the grid take care of the flow of new grid items and the generation of new cells.

Q5 Explain why minimizing server requests is a good web development practice, and provide an example of how a web developer may minimize server requests.

Minimizing server requests on web pages is a good web development practice because it enhances web page performance. Minimizing server requests also helps to reduces the workload on the server, enhancing performance as busy servers may be slower to send and receive data.

Each HTTP request to a server takes time. When visiting the web page, there is the initial HTTP request from the browser to the server. If there are elements in the HTML document like images, linked CSS files, or linked JavaScript files, the browser sends additional HTTP requests to the server to access the resource. These HTTP requests take time, and minimizing server requests reduces the time it takes to assemble the web page.

A web developer may minimize server requests by embedding CSS and JavaScript into web pages. Rather than linking to external files that send additional HTTP requests, a web developer may use the <style> and <script> tags to embed CSS rules and JavaScript directly into the HTML document.
