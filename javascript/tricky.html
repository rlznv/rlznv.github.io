Troubleshooting Methods

Test in Different Browsers

To make sure your code is working properly, testing your code in different browser types can help determine errors.

Review Your Code

If your code isn't working, try asking someone to review your code. A pair of fresh eyes may be able to catch errors you've missed.

If you can't find anyone to review your code, try rubber duck debugging. Pretending to explain your code, line-by-line, to someone who understands nothing about the code may help you run through the logic of your code and find potential errors.

Lint Your Code

Using a JavaScript linter (a linting program) is an easy way to troubleshoot your JavaScript code.

Like using a lint roller on clothes, a linter is a program that analyzes code to catch potential errors. You can use programs like JSLint or ESLint to analyze your JavaScript code.

Use an IDE with Debugging Features

Some IDEs (integrated development environments), Visual Studio Code, for example, comes with built-in debugging features. These debugging features can catch errors in your code as you're writing it and during the code's execution.

If your code isn't working and your IDE's showing errors and warnings, that may explain your problem.

JavaScript Libraries

You may have heard about the power of JavaScript libraries and frameworks that can take your projects to the next level. However, it's important to keep in mind that libraries and frameworks aren't exactly the same thing.

JavaScript libraries are pre-written JavaScript code that can be integrated into existing projects to provide visual flair, functionality, and interactive behavior to web pages.

On the other hand, JavaScript frameworks are structured, comprehensive tools that can be used to build web pages and applications. JavaScript frameworks are much more comprehensive than libraries, providing a structured way to build both the front ends and back ends of web pages and applications.

Some popular JavaScript libraries include:

  • jQuery
  • Bootstrap
  • ReactJS

For this class, you will touch on JavaScript only a small portion of the class. If you include JavaScript, it will most likely be pulled from JavaScript libraries rather than hand-writing the code. It is important to understand the anatomy of JavaScript enough to recogonize it and what it does when you come across it.
